Loom documentation

Loom is a package manager for AI skills. It installs, inspects, and authors reusable skills for AI agents such as Claude Code, OpenAI Codex, and Cursor.

A skill is a folder of instructions and helper files an agent loads to do a job well — writing PDFs, building slide decks, designing artifacts, and so on. Loom does for those skills what Homebrew does for command-line tools: a shared repository of manifests describes where each skill lives and how to fetch it, and Loom weaves the payload into your agent's skills folder on request.

A manifest never contains the skill payload itself. It only describes how to obtain and place a skill, in the spirit of a Homebrew formula. That keeps the repository small and lets skills live in their own upstream projects.

Installation

Loom is a single Rust binary. Install it with Cargo:

cargo install loom

Confirm it is on your PATH and check your environment:

loom --version
loom doctor

loom doctor checks for common problems — a reachable skill repository, writable agent skills folders, and the tools Loom relies on to fetch sources.

Quickstart

Find a skill, install it, and confirm it landed.

# search the repository
loom search pdf

# read the details before installing
loom info pdf

# install into the default agent (Claude Code)
loom install pdf

# see what's installed
loom list

By default Loom installs into Claude Code at ~/.claude/skills. Target a different agent with --agent: 

loom install pdf --agent codex
Agents Loom knows about: claude-code (~/.claude/skills, the default), codex (~/.codex/skills), and cursor (~/.cursor/skills). Run loom agents to list them and see which is the default.

CLI reference

Every command Loom ships with. Run loom <command> --help for full flags.

CommandWhat it does
loom install <skill>Install a skill for an agent. Accepts a repo skill name, or a path/URL to a manifest. Use --agent to target a specific agent.
loom remove <skill>Remove an installed skill. Aliases: uninstall, rm.
loom listList installed skills. Alias ls. Filter with --agent.
loom search <query>Search the skill repository by name, description, and keywords.
loom info <skill>Show full details for a skill from its manifest.
loom updateSync the local skill repository from its remote.
loom upgrade [skill]Upgrade installed skills to the repository's version. Omit the name to upgrade all.
loom new <name>Scaffold a new manifest to author from. Use --out to choose where it's written.
loom lint [path]Validate a manifest file, or every manifest in the repository when no path is given.
loom test <skill>Fetch and stage a skill without installing it, to verify its manifest resolves.
loom generate <url>Generate manifests by inspecting a skills repository URL (alias: create; URL may also be passed as --url). Pin with --ref, write with --out.
loom indexBuild the website search index from the repository. Defaults to docs/skills.json; override with --out.
loom agentsList the agents Loom can install into and their skills folders.
loom doctorCheck the environment for common problems.

Manifest format

A manifest is a YAML file named <skill_name>.yml that lives in a Loom repository's skills/ folder. It describes how to fetch and lay down one skill. 

name: pdf
version: 0.1.0
description: Read, fill, and generate PDF documents.
homepage: https://github.com/anthropics/skills
license: MIT
authors:
  - Anthropics <skills@anthropic.com>
keywords:
  - pdf
  - documents
compatibility:
  - claude-code
  - codex
source:
  type: git            # git | archive
  url: https://github.com/anthropics/skills
  ref: v0.1.0          # git tag, branch, or commit (git sources)
  # sha256: <hash>     # expected archive checksum (archive sources)
  subdir: pdf          # folder inside the repo that is the skill
install:
  entry: SKILL.md      # the file an agent loads first
  files: []            # explicit paths to include; empty copies everything

Fields

FieldMeaning
nameUnique skill identifier. Lowercase.
versionSemver-ish version string.
descriptionOne-line summary shown in search and info.
homepageProject homepage URL.
licenseSPDX license identifier. Optional.
authors[]Free-form Name <email> strings.
keywords[]Search tags.
compatibility[]Agents the skill is known to work with.
source.typegit to clone a repo, or archive to download a .tar.gz.
source.urlClone or download URL.
source.refGit tag, branch, or commit. Git sources only.
source.sha256Expected checksum of the archive. Archive sources only.
source.subdirSub-directory inside the repo/archive that holds the skill. Empty means the root is the skill.
install.entryThe file an agent loads first. Defaults to SKILL.md.
install.files[]Explicit relative paths to include. Empty copies everything.

Authoring a skill

Contributing a skill is scaffold, edit, validate, test, and open a pull request.

  1. Scaffold a manifest with loom new.
  2. Edit the generated <name>.yml — point source.url / subdir at where the skill actually lives, and fill in the metadata.
  3. Lint it to catch missing or malformed fields.
  4. Test it — Loom fetches and stages the payload without installing, proving the source resolves.
  5. Open a pull request that adds the manifest to the repository's skills/ folder.
loom new my-skill
# edit skills/my-skill.yml — set source.url, subdir, description…
loom lint skills/my-skill.yml
loom test my-skill

When lint and test pass, commit the manifest under skills/ and open a PR against the repository. Once it merges and the index is rebuilt, your skill is installable with loom install my-skill.

Have an existing skills repository? loom generate <url> inspects it and drafts manifests for you — a fast way to bring many skills into Loom at once.

How the website index is built

This site's search is powered by a generated index at docs/skills.json. It is produced by reading every manifest in the repository's skills/ folder and writing out their metadata: 

loom index
# defaults to docs/skills.json; override the destination:
loom index --out docs/skills.json

The output is a small JSON document the Browse page fetches at runtime:

{
  "generated_by": "loom 0.1.0",
  "count": 17,
  "skills": [
    {
      "name": "pdf",
      "version": "0.1.0",
      "description": "Read, fill, and generate PDF documents.",
      "homepage": "https://github.com/anthropics/skills",
      "license": null,
      "authors": ["anthropics"],
      "keywords": [],
      "compatibility": ["claude-code"],
      "source": "https://github.com/anthropics/skills"
    }
  ]
}

Because GitHub Pages serves the docs/ folder as-is, re-running loom index and committing the updated skills.json is all it takes to refresh what appears on the Browse page.
